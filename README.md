# ScaRigisterExtension

给悬镜 SCA 使用的 Chrome 批量注册扩展。

仓库只保留两类内容：

- `chrome-extension/`：可直接加载到 Chrome 的扩展目录
- `template/SCA用户账号模板.xlsx`：空白导入模板

## 功能

- 从当前已登录的悬镜 SCA 页面自动读取站点地址和登录态
- 本地填写默认密码、SM2 公钥或固定密文
- 导入 Excel 模板后批量创建用户
- 自动查询并匹配部门 ID
- 自动查询并匹配项目 ID
- 对重名或模糊命中的部门/项目提供下拉确认
- 项目可留空，提交时会自动传 `projectMemberList: []`

## 目录结构

```text
ScaRigisterExtension/
├── chrome-extension/
└── template/
    └── SCA用户账号模板.xlsx
```

## 使用方式

### 1. 加载扩展

1. 打开 Chrome 扩展管理页：`chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择 `chrome-extension/` 目录

### 2. 准备页面登录态

1. 在浏览器里打开已登录的悬镜 SCA 页面
2. 点击扩展图标
3. 先点一次“刷新认证”

扩展会自动读取：

- 当前页面 `origin` 作为目标站点
- 当前页面里的 `Token`

## 3. 配置本地参数

下面这些值只保存在浏览器本地，不写入仓库：

- `默认密码`
  对应：统一初始密码明文
- `SM2 公钥（本地配置）`
  对应：当前环境使用的 SM2 公钥
- `固定加密密码（可选）`
  对应：页面成功请求里的 `password` 密文
  填了之后会直接复用，不再现场加密
- `默认角色 ID`
  普通用户一般为 `1`
- `兜底部门`
  当模板里的部门没匹配到系统部门时使用
- `创建接口路径`
  一般保持 `/sca/api-v1/user/add`

## 4. 导入模板

模板文件在 `template/SCA用户账号模板.xlsx`。

填写规则：

- 必填列：`用户名`、`真实姓名`、`邮箱`、`手机号`、`所属部门`
- 选填列：`所属项目`
- `所属项目` 可以留空
- 不要修改表头和说明行

## 5. 确认映射并执行

导入 Excel 后：

- 部门会按唯一部门名自动匹配
- 项目会按唯一项目名逐个调用 `/sca/api-v1/project/list` 做查询
- 只有唯一精确命中才会自动选中
- 命中多个候选时，需要你在 popup 里手动确认
- 项目不想关联时，直接留空即可

确认完成后，点击“开始批量创建”。

## 注意事项

- 公开版仓库不会内置默认密码
- 公开版仓库不会内置 SM2 公钥
- 如果后端不接受明文密码，必须配置 `SM2 公钥` 或 `固定加密密码`
- 修改扩展代码后，需要在 Chrome 扩展页先 `Reload`，再回 SCA 页面强刷

## 适用范围

本扩展面向悬镜 SCA 的 Web 管理页面，依赖当前页面登录态和页面接口行为。不同版本的 SCA 若接口字段变化，需要按实际页面请求调整。
